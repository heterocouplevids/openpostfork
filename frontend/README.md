# OpenPost Frontend

SvelteKit frontend for [OpenPost](../README.md) — a lightweight, self-hosted social media scheduler with web and Android app support.

## Tech Stack

- **SvelteKit 5** with runes (`$state`, `$derived`, `$effect`)
- **TailwindCSS 4** for styling
- **shadcn-svelte** UI component library
- **Paraglide** for i18n (English, Portuguese)
- **openapi-fetch** for typed API client (generated from backend OpenAPI spec)
- **Vitest** for unit testing
- **Capacitor** for Android mobile app support

## Project Structure

```
frontend/
├── src/
│   ├── lib/
│   │   ├── api/           # Typed API client + generated types
│   │   ├── components/    # UI components (app + shadcn primitives)
│   │   ├── stores/        # Svelte stores (auth, UI, Mastodon instance)
│   │   ├── paraglide/     # Auto-generated i18n runtime
│   │   └── types/         # Shared TypeScript types
│   └── routes/
│       ├── +page.svelte   # Dashboard/home
│       ├── accounts/      # Connected accounts management
│       ├── connect/       # Connect social platforms
│       ├── login/         # Login page
│       └── register/      # Registration page
├── android/               # Native Android project (Capacitor)
├── messages/              # Paraglide i18n source files (en, es, pt)
├── openapi.json           # OpenAPI spec from backend (used for type generation)
└── capacitor.config.ts    # Capacitor config for Android build
```

## API Client

The typed API client is generated from the backend's OpenAPI spec:

1. Backend exposes `/openapi.json` (generated by Huma)
2. Copy to `frontend/openapi.json`
3. Run `pnpm run generate:types` to regenerate `src/lib/api/types.d.ts`
4. Client in `src/lib/api/client.ts` uses `openapi-fetch` for type-safe API calls

## i18n

Translations are managed with Paraglide. Source messages are in `messages/*.json`.
Supported languages: English (`en`), Portuguese (`pt`). Spanish
(`es.json`) was previously a stub; it has been removed until a real
translation is contributed (ROADMAP item 10).

To add translations:

1. Edit the relevant `messages/{lang}.json` file
2. Run `pnpm dlx @inlang/cli machine translate` (or edit manually)
3. The Paraglide runtime is auto-generated in `src/lib/paraglide/`

## Android (Capacitor)

The app is built as a native Android wrapper around the same SvelteKit
frontend the web build uses. A pre-built APK ships with every GitHub
release — see the [Android app docs](../../docs-site/installation/android.md)
for install and configuration.

### Local debug build

```bash
pnpm run build:capacitor   # vite build + cap sync android + assets
cd android
./gradlew assembleDebug
# APK at android/app/build/outputs/apk/debug/app-debug.apk
```

### Local signed release build

```bash
cd android
./gradlew assembleRelease \
    -PRELEASE_STORE_FILE=path/to/release.keystore.jks \
    -PRELEASE_STORE_PASSWORD=... \
    -PRELEASE_KEY_ALIAS=... \
    -PRELEASE_KEY_PASSWORD=...
```

Branding assets (launcher icon, splash screen) are generated automatically
from `assets/brand/icon.svg` during `pnpm run build:capacitor`, so the
Android app stays visually consistent with the web app and PWA.

For a deeper look at the Android configuration, the Gradle variables
(`android/variables.gradle`), the bundled Capacitor plugins
(`@capacitor/app`, `@capacitor/splash-screen`, `@capacitor/status-bar`),
and how the release workflow packages the APK, see the
[Android app docs](../../docs-site/installation/android.md).

## Development

The frontend can run independently against a local backend:

```bash
pnpm run dev
```

The dev server runs on `http://localhost:5173` and proxies API requests to the backend at `http://localhost:8080`.
